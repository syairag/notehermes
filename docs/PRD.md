## 1. 产品概述

### 1.1 产品定位
**NoteHermes** 是一款 AI 驱动的智能工作台，深度整合**邮箱、笔记、任务**三大核心模块。
区别于传统工具（Notion/飞书/Obsidian），NoteHermes 的 Agent 不是被动问答机，而是**主动连接数据流、自动提取行动项的智能参谋**。

> **Slogan：** "邮件自动变任务，笔记自动变知识，任务自动被跟进。"

### 1.2 目标用户
- 职场管理者/老板：信息过载，需要从海量邮件和文档中快速提取行动项
- 知识工作者：频繁切换邮件、笔记、任务工具，效率被上下文切换消耗
- 效率工具爱好者：追求"打开即掌控"的一体化体验

### 1.3 核心价值主张
- **打通数据孤岛：** 邮件 → 笔记 → 任务，三模块数据天然互通
- **零手动提取：** 从长邮件中自动提炼待办，从会议记录中自动识别行动项
- **活的知识库：** 笔记不再是死的文档，而是能被语义搜索、关联推荐、自动更新的"活资产"

## 2. Phase 1 MVP 功能规划（4-6 周）

### 2.1 总体策略：先轻后重
**Phase 1 核心目标：** 做透"连接"，让 Agent 当辅助。不碰"自主执行"，先把数据管道和用户体验跑通。

### 2.2 📧 邮箱模块

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 多邮箱接入 | 支持 IMAP/SMTP 接入主流邮箱（Gmail/Outlook/企业邮箱/飞书邮箱），**特别支持国内世纪互联版 Microsoft 365 企业邮箱** | 🔴 P0 |
| AI 邮件摘要 | 长邮件自动生成 3 行摘要，一眼看清核心内容 | 🔴 P0 |
| 待办自动提取 | Agent 识别邮件中的"待办意图"（如"周五前发我方案"），生成任务建议 | 🔴 P0 |
| 智能分类 | 自动按项目/紧急度分类邮件，减少手动整理 | 🟡 P1 |
| 快捷回复 | AI 根据历史邮件和笔记内容，生成回复草稿 | 🟢 P2 |

### 2.3 📝 笔记模块

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 快速笔记 | 支持 Markdown，一键新建，秒级响应 | 🔴 P0 |
| 跨笔记语义搜索 | 基于向量检索，"找东西一秒到位"，不再依赖关键词匹配 | 🔴 P0 |
| 自动标签分类 | Agent 根据内容自动打标签（项目/客户/技术方向等） | 🟡 P1 |
| 邮件上下文关联 | 笔记可关联特定邮件，点击跳转查看原文 | 🟡 P1 |
| 知识图谱预览 | 展示笔记间的关联关系（Phase 1 简化版：标签共现 + 链接推荐） | 🟢 P2 |

### 2.4 ✅ 任务模块

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 自动创建任务 | 从邮件/笔记中提取的行动项，一键或自动创建为任务 | 🔴 P0 |
| 智能排期建议 | Agent 根据任务内容、紧急度、历史习惯，推荐截止日期和时间块 | 🟡 P1 |
| 任务看板/列表 | 支持看板视图（To Do / Doing / Done）和列表视图 | 🔴 P0 |
| 任务 ↔ 笔记关联 | 完成任务时可自动关联相关笔记；打开笔记时推荐未完成任务 | 🟡 P1 |
| 提醒与通知 | 桌面通知、推送通知、定时提醒 | 🔴 P0 |

### 2.5 🔗 核心连接（NoteHermes 的灵魂）

| 数据流 | 说明 | 优先级 |
|--------|------|--------|
| 📧→✅ 邮件变任务 | 邮件中的行动项自动/半自动转为任务 | 🔴 P0 |
| 📝→✅ 笔记变任务 | 笔记中的"@某人"、"TODO"、"待办"自动提取为任务 | 🟡 P1 |
| ✅→📝 任务关联笔记 | 任务详情页自动展示相关笔记、历史上下文 | 🟡 P1 |
| 📧→📝 邮件沉淀为笔记 | 重要邮件一键保存为笔记，保留原始格式和附件 | 🟢 P2 |

## 3. 非功能性需求

| 维度 | 要求 |
|------|------|
| **性能** | 首屏加载 < 2s，AI 摘要响应 < 5s，搜索响应 < 1s |
| **隐私** | 邮件和笔记数据本地加密存储，AI 处理采用脱敏后调用或本地模型 |
| **离线可用** | 核心功能（笔记查看、任务管理）支持离线操作，联网后自动同步 |
| **跨平台** | 首发：HarmonyOS NEXT + Android；后续扩展：Web/PWA、macOS/Windows |

## 5. 风险评估

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| AI 模型 API 延迟/不稳定 | 摘要和搜索体验差 | 采用本地小模型兜底 + 云端大模型降级策略 |
| 邮箱协议兼容性 | 部分企业邮箱无法接入 | Phase 1 先覆盖主流 IMAP/SMTP，特殊协议后续适配 |
| 隐私合规 | 用户对数据上传 AI 有顾虑 | 明确数据脱敏流程，支持纯本地模式（功能受限但安全） |
| 开发周期失控 | 6 周交付压力大 | 严格砍 P2 功能，聚焦 P0 核心链路 |

## 6. Phase 2 前瞻（重 Agent，7-12 周）

- Agent 自主执行任务（调研、竞品分析、报告生成）

- 定时抓取行业资讯，推送个性化日报
- 知识图谱 2.0：自动发现笔记间的隐性关联
