## 2. 核心技术选型

### 2.1 客户端
| 平台 | 技术栈 | 说明 |
|------|--------|------|
| HarmonyOS NEXT | ArkTS + ArkUI | 原生体验，利用鸿蒙分布式能力 |
| Android | Kotlin + Jetpack Compose | 标准 Android 开发 |
| Web / PWA | Next.js 14 + TailwindCSS | 快速迭代，覆盖桌面端用户 |

**跨端策略：** 核心业务逻辑封装为独立的 TS/Rust SDK，各端共享。UI 层按平台原生开发。

### 2.2 后端服务
| 组件 | 选型 | 理由 |
|------|------|------|
| API 框架 | FastAPI (Python) / Go (Gin) | Python 适合 AI 集成；Go 适合高并发 API 网关 |
| 数据库 | PostgreSQL 15 | 成熟稳定，支持 JSONB + 事务 |
| 向量检索 | pgvector (PG 扩展) | 与主库统一，简化运维，Phase 1 足够 |
| 缓存/队列 | Redis 7 | 缓存、会话、消息队列 |
| 对象存储 | MinIO / S3 兼容 | 附件、图片、语音文件存储 |

### 2.3 AI 层
| 模块 | 模型/技术 | 说明 |
|------|----------|------|
| 邮件摘要/提取 | Qwen-Plus / GPT-4o-mini | 高性价比，适合结构化提取 |
| 笔记语义搜索 | text-embedding-3-small / bge-m3 | 向量嵌入，支持多语言 |
| 本地兜底模型 | Qwen2.5-7B (llama.cpp) | 断网/隐私敏感时降级使用 |
| Agent 编排 | LangChain / 自研 Prompt Pipeline | Phase 1 以 Prompt 工程为主 |

## 3. 核心数据流设计

### 3.1 📧 邮件 → ✅ 任务 提取流程

```
IMAP 收件
    │
    ├─→ 邮件解析（发件人/主题/正文/附件元数据）
    │
    ├─→ AI 意图识别（Prompt: "这封邮件是否包含待办事项？"）
    │       ├─ 否 → 仅生成摘要，存入邮箱
    │       └─ 是 → 提取以下信息：
    │              • 任务描述
    │              • 截止时间（如有）
    │              • 优先级（紧急/普通/低）
    │              • 关联人（如有）
    │
    ├─→ 生成"任务建议"卡片（用户确认 or 自动创建）
    │
    └─→ 任务入库 + 通知用户
```

### 3.2 📝 笔记语义搜索流程

```
用户输入查询
    │
    ├─→ 查询文本 → Embedding 模型 → 向量
    │
    ├─→ pgvector 相似度搜索（Top-K）
    │
    ├─→ RAG 增强：检索结果 + 原始查询 → AI 重写/排序
    │
    └─→ 返回关联笔记列表 + 高亮匹配片段
```

### 3.3 跨模块关联存储

```sql
-- 统一实体关联表（核心设计）
CREATE TABLE entity_links (
    id UUID PRIMARY KEY,
    source_type VARCHAR(20),  -- 'email', 'note', 'task'
    source_id UUID,
    target_type VARCHAR(20),
    target_id UUID,
    link_type VARCHAR(20),    -- 'extracted_from', 'related_to', 'attachment'
    metadata JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 4. 隐私与安全设计

### 4.1 数据分级
| 级别 | 数据类型 | 处理方式 |
|------|---------|---------|
| 🔴 敏感 | 邮件正文、笔记内容、密码 | 本地加密存储，AI 调用时脱敏（去标识化） |
| 🟡 内部 | 任务状态、标签、元数据 | 服务端加密传输 |
| 🟢 公开 | 用户头像、显示名 | 标准 HTTPS 传输 |

### 4.2 AI 调用隐私策略
- 邮件/笔记内容发送给 AI 前，自动替换真实姓名/邮箱/电话为占位符
- 支持"纯本地模式"：所有 AI 推理在设备端完成（功能受限但 100% 隐私）
- 不存储 AI 调用的原始 Prompt 和 Response 日志
